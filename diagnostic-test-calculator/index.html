Diagnostic Test Calculator: Calculate Sensitivity, Specificity and Predictive Value

In the era of evidence-based medicine (EBM), the simple concept of “test accuracy” is not enough. To correctly interpret medical tests, screenings, and machine learning algorithms, it is necessary to understand how the test behaves in sick and healthy patients.

Our diagnostic value calculator allows you to instantly turn raw data (true and false positive/negative results) into professional metrics: sensitivity, specificity, predictive value and likelihood ratios.

How to use the calculator: Error matrix

The basis of the calculation is Confusion Matrix - a 2x2 table that compares test results with the patient's actual status (usually confirmed by the Gold Standard).

To get the result you need to fill in four fields:

  1. TP (True Positive): The patient objectively sick, and the test is true confirmed.
  2. FP (False Positive): Patient healthy, but the test showed erroneously disease. In statistics, this is called a Type I error or “false alarm.”
  3. FN (False Negative): Patient sick, but the test showed erroneously health. This is a type II error or “missing the target”, the most dangerous in medicine.
  4. TN (True Negative): Patient healthy, and the test is true confirmed.

Advanced Metrics: Likelihood Ratio

Clinicians and researchers often use the Likelihood Ratio (LR) because these measures do not depend on the prevalence of the disease in a particular population, unlike PPV and NPV.

LR+ (Positive Likelihood Ratio)

Shows how many times the probability of getting a positive test is higher in a sick person than in a healthy one.

  • Formula: $$\text{Sens} / (1 — \text{Spec})$$
  • Value > 10 — the test result is a strong confirmation of the diagnosis.
  • Value ≈ 1 - the test is useless, it does not change the likelihood of disease.

LR- (Negative Likelihood Ratio)

Shows how many times the probability of receiving a negative test is higher in a sick person than in a healthy person.

  • Formula: $$(1 — \text{Sens}) / \text{Spec}$$
  • Value < 0.1 — the test result allows us to exclude the diagnosis with high confidence.

Example of calculating test efficiency

Let's imagine that we are testing a new rapid method for diagnosing influenza on a group of 200 patients. After comparison with laboratory analysis (PCR), we obtained the following data:

  • TP = 50 (Sick with flu, test positive)
  • FP = 10 (Healthy, test positive)
  • FN = 5 (Sick with flu, test negative)
  • TN = 135 (Healthy, test negative)

After entering this data into the calculator we get the following picture:

  • Sensitivity: 90.9% (The test successfully finds 9 out of 10 patients).
  • Specificity: 93.1% (The test rarely gives false alarms in healthy people).
  • PPV: 83.3% (If the patient received a “plus”, the risk that it is really the flu is 83%).
