Technical features of the conversion

The AVIF to PNG conversion process involves decoding the original compressed stream into an array of raw pixels inside a Canvas element, then applying the classic PNG packaging algorithm.

Calculation of image volume in RAM

It is important to understand that no matter how highly the AVIF file is compressed on disk, when it is opened in the browser’s RAM, the uncompressed image takes up a strictly defined amount of space. For RGBA images, the memory size is calculated using the following formula:

$$Size = W \times H \times 4$$

Where $W$ is the width of the image in pixels, $H$ is the height, and $4$ is the number of bytes allocated per pixel (Red, Green, Blue and Alpha channels).

Scaling when exporting

Built-in converter settings allow you to change the size of the resulting file on the fly. If you set the scale to $S$ (as a percentage of the original), the new dimensions are calculated linearly, rounded to the nearest pixel:

$$W_{new} = \lfloor W_{old} \times \frac{S}{100} \rceil$$

$$H_{new} = \lfloor H_{old} \times \frac{S}{100} \rceil$$

This function allows you not only to change the graphics format, but also to optimize the physical resolution of the image before saving if the source file turns out to be excessively large.
