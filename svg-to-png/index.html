Online SVG to PNG converter: batch rasterization, transparent background and scaling

Mathematical model of scaling and rendering

Our converter uses the rendering library onCanvas, which mathematically maps each vector node into a pixel matrix. When aspect ratio scaling is enabled, the final pixel resolution of the image ( $W_{png}$ and $H_{png}$ ) is calculated based on the original aspect ratio of the vector ( $Ratio$ ) and the specified multiplier ( $S$ ):

$$W_{png} = \lfloor W_{svg} \times S \rfloor, \quad H_{png} = \lfloor \frac{W_{svg} \times S}{Ratio} \rfloor$$

The computational complexity of the rasterization time ( $T_r$ ) depends on both the number of mathematical Bezier curves ( $P$ ) in the source SVG file and the final area of ​​the generated raster ( $A = W_{png} \times H_{png}$ ). The process is described by the asymptotic complexity formula:

$$T_r = \mathcal{O}(P) + \mathcal{O}(W_{png} \times H_{png})$$

This means that our algorithm first builds the geometry and then fills the pixels. By optimizing browser API calls, even the most complex illustrations with gradients and hundreds of layers are processed with maximum performance.
