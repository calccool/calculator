Technical Comparison of Data Standards

To understand how the conversion occurs, it is important to consider the structural differences between both formats. Both file types are archives containing XML documents, but their architecture is different.

Evaluation of compression and processing efficiency

When the source XML tree is converted to the Office Open XML format, the data is repacked using the Deflate algorithm. The change in final file size can be predicted using a basic packaging efficiency ratio:

$$C_{ratio} = \left( 1 — \frac{S_{xlsx}}{S_{ods}} \right) \times 100\%$$

Where $C_{ratio}$ is the percentage of resizing, $S_{xlsx}$ is the size of the resulting Excel file in bytes, and $S_{ods}$ is the original size of the OpenDocument table. In most cases, XLSX exhibits higher compression density for text nodes.
