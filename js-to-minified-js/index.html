The Mathematics of Compression: How We Measure Efficiency

The minification process is based on lexical analysis algorithms. The efficiency of the minifier (compression ratio) can be expressed by a strict mathematical formula for estimating the saved space:

$$C = \left( 1 — \frac{V_{min}}{V_{orig}} \right) \times 100\%$$

Where the variables mean the following:

  • $C$ - final compression percentage (saved space).
  • $V_{orig}$ - size of the source file in bytes before processing.
  • $V_{min}$ - file size.min.jsafter passing through the AST parser.

The higher the $C$ score, the faster the client browser will download, parse and execute your script, which directly affects the First Input Delay (FID) metric.

How does the conversion process work?

Turning readable code into compressed code.min.jsa file is a complex multi-step process. If we break down the work of our parser into stages, the algorithm looks like this:

  1. Tokenization: The entire text is divided into the smallest semantic units (tokens) - keywords, operators, parentheses.
  2. Building a tree: An abstract syntax tree (AST) is assembled from tokens. At this stage, the tool checks the code for syntax errors.
  3. Traversal and Transformation (Mangling): The engine goes around the tree. Long local variable names (for example,calculateTotalSum) are replaced by short ones (for example,x) if it is scope-safe.
  4. Code generation: The optimized tree turns back into text, but without spaces, line breaks and comments.
