Online IPv4 and IPv6 subnet mask calculator - Calculate CIDR and IP ranges

A subnet mask is a 32-bit (for IPv4) or 128-bit (for IPv6) number that divides an IP address into two functional parts: the address of the network itself and the address of a specific node (host) within that network. The use of subnets is critical to effectively manage traffic, ensure security, and prevent address space exhaustion.

Our subnet mask calculator allows you to instantly calculate network boundaries using both classic decimal notation and the modern CIDR (Classless Inter-Domain Routing) standard.

Mathematical logic for calculating IPv4

To determine network parameters, the calculator uses bitwise operations on the IP address and mask. The main parameters are calculated using the following formulas:

1. Number of hosts on the network:

If $n$ is the number of bits allocated to the host address (32 minus the CIDR prefix), then the total number of $N$ addresses is calculated as:

$$N = 2^n$$

The available number of Usable Hosts is usually 2 less, since the first address is reserved for networking and the last for broadcasting:

$$N_{usable} = 2^{(32 — CIDR)} — 2$$

2. Network Address:

Obtained by performing a bitwise AND operation between the IP address and the subnet mask.

3. Wildcard mask:

This is the inverse of a regular subnet mask. It is calculated by subtracting each octet of the mask from 255:

$$W = 255.255.255.255 — Mask$$

Features of calculating IPv6 subnets

In the IPv6 protocol, the address space is much wider ( $128$ bit vs. $32$ ). The concept of a broadcast address (Broadcast) and a mask in decimal form are not used here.

  • Standard prefix: In most cases, the prefix used for end networks is/64.
  • Network capacity: Network with prefix/64contains $2^{64}$ addresses, representing more than $18$ quintillion devices.
  • Hierarchy: The calculator allows you to calculate how many standard subnets/64can be isolated from a larger block (for example, from/48or/56).
