The Domain Name System (DNS) was designed decades ago and only supports a limited set of characters. ASCII (Latin alphabet, numbers and hyphen). With the advent of multilingual domains ( IDN - Internationalized Domain Names), such as president.rf or shop.online, there was a need to “explain” to the old servers how to work with Cyrillic.

Punycode is a specialized transformation algorithm that converts Unicode characters into a sequence of ASCII characters. Any Cyrillic domain on the network is technically presented in the format ACE (ASCII Compatible Encoding), which always starts with a prefixxn--.

How the conversion algorithm works

The coding process is based on the method Bootstring, which effectively separates base characters from extended characters and encodes their positions. Mathematically, this can be represented as a function to convert the character set $U$ (Unicode) to the string $A$ (ASCII):

$$A = \text{Punycode}(U) \rightarrow \text{«xn--» } + \text{encoded\_string}$$
