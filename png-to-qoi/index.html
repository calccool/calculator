The Secret to Performance: The Mathematics of the Algorithm

Unlike PNG, which uses dictionaries and block encoding, QOI works on the principle of stream hashing previously seen pixels. This allows the processor to process data with a minimum number of clock cycles.

Searching an array of 64 colors (hash table) is carried out using a lightweight indexing formula:

$$index = (r \times 3 + g \times 5 + b \times 7 + a \times 11) \pmod{64}$$

Where the variables denote the color channels of the current pixel:

  • $r$ - red channel (Red)
  • $g$ - green channel (Green)
  • $b$ - blue channel (Blue)
  • $a$ - transparency (Alpha)

This approach eliminates the need for the system to build complex Huffman trees. The efficiency of the final compression (Compression Ratio) can be calculated using the classic formula:

$$CR = \frac{Size_{raw}}{Size_{qoi}}$$

Where $Size_{raw}$ is the size of the raw uncompressed raster (Width × Height × 4 bytes), and $Size_{qoi}$ is the final file size. Practice shows that for most images this coefficient is almost identical to that of PNG, and the time gain is enormous.
