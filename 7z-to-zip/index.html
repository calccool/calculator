Mathematics of Compression: Performance Analysis

When repacking from 7Z to ZIP, the size of the final file will increase slightly, since the Deflate algorithm is inferior to LZMA in packing density. The effectiveness of any archiving algorithm is assessed through the compression ratio.

The following mathematical model is used for assessment:

The compression ratio ( $CR$ ) is calculated as the ratio of the volume of the original data to the volume of the archive:

$$CR = \frac{V_{uncompressed}}{V_{compressed}}$$

Where:

  • $V_{uncompressed}$ - original data size (before compression).
  • $V_{compressed}$ - size of the received archive.

The percentage of disk space saved ( $S$ ) is calculated using the formula:

$$S = \left( 1 — \frac{1}{CR} \right) \times 100\%$$

What does this look like in practice?

If you are compressing a document folder of size 100 MB:

  1. The 7Z algorithm (LZMA2) can compress it to 20 MB ( $CR = 5$ , save $80\%$ ).
  2. The ZIP (Deflate) algorithm will compress it to approximately 35 MB ( $CR \approx 2.85$ , save $65\%$ ).

Despite the slight loss in compression ratio when moving from 7Z to ZIP, you get a 100% guarantee that the recipient will be able to open your archive on any device without installing additional programs.
