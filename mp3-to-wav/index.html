Technical algorithms for audio processing

Our tool not only changes the container, but also allows you to improve the basic characteristics of the audio track. When you activate the “Loudness Normalization” option, the system analyzes the entire audio file for the presence of peaks and proportionally amplifies quiet sections.

The mathematical model for calculating the gain to normalize to −1 dBFS looks like this:

$$M = \frac{10^{\frac{-1}{20}}}{P_{max}}$$

Where $M$ is the resulting multiplier for each audio sample, and $P_{max}$ is the maximum absolute amplitude value found in the source file data array. This allows you to extract quiet voice recordings without allowing digital overload (clipping).

Predicting the final file size

Since WAV is an uncompressed format (PCM), the final file size after conversion can be calculated mathematically to the nearest byte before rendering begins. The memory allocation algorithm is based on the following formula:

$$S_{wav} = \left( \frac{R_s \times D_b \times C}{8} \right) \times T + 44$$

In this equation:

  • $S_{wav}$ - final file size in bytes.
  • $R_s$ - selected sampling frequency (for example, 48000 Hz).
  • $D_b$ - bit depth (our converter generates standard 16-bit samples).
  • $C$ - number of channels (1 for mono, 2 for stereo).
  • $T$ - the exact duration of the audio file in seconds.
  • Constant 44 is the size of the standard RIFF/WAVE header in bytes.

This strict approach to creating an audio container ensures that the resulting WAV files will be read correctly in any professional software, from Adobe Premiere Pro to FL Studio.
