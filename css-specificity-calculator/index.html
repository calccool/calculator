CSS Specificity Calculator: Online Calculation of Selector Weights

Specificity is a set of rules that the browser uses to determine which CSS property will be applied to an element if it is affected by multiple conflicting rules. This is a kind of "weight" or hierarchy of importance of selectors.

Developers often encounter a situation where a written style is not applied to an element. In 90% of cases the reason is that another selector has higher priority. Our calculator allows you to instantly calculate this priority, eliminating the need to use!importantand litter the code.

How does the weight calculation formula work?

The selector weight is represented as a tuple of four numbers: $(a, b, c, d)$ . The further to the left the number in this sequence, the more significant it is.

Mathematically, this can be represented by the following formula for the conditional “decimal” representation of weight:

$$TotalWeight = (a \times 1000) + (b \times 100) + (c \times 10) + (d \times 1)$$

Where variables mean:

  • $a$ - Availability of inline styles.
  • $b$ — Number of identifiers (ID).
  • $c$ — Number of classes, attributes and pseudo-classes.
  • $d$ — Number of element names (tags) and pseudo elements.

Specificity Calculation Examples

Let's look at a few classic examples that are often found in real practice:

  1. Selector:div.container
    • Composition: 1 tag (div) + 1st grade (.container).
    • Result: $(0, 0, 1, 1)$ . Decimal weight: 11.
  2. Selector:#sidebar .menu-item:hover
    • Contents: 1 ID (#sidebar) + 1st grade (.menu-item) + 1 pseudo-class (:hover).
    • Result: $(0, 1, 2, 0)$ . Decimal weight: 120.
  3. Selector:body header nav ul li a
    • Composition: 6 tags.
    • Result: $(0, 0, 0, 6)$ . Decimal weight: 6.

Important Exceptions

  • Universal selector (*), combinators (+, >, ~) and the negation pseudo-class (:not()) on their own have no weight (0, 0, 0, 0). However, the selectors inside:not()are considered as usual.
  • !important is not part of specificity, but a “terminator” of the cascade. It overrides any calculations, but its use is considered bad practice.
